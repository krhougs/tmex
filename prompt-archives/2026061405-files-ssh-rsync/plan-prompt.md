# Files Tab — SSH/rsync 支持与体验优化（Prompt 存档）

承接 `2026061404-files-tab`（本地文件浏览已完成）。本轮扩展到 SSH 设备 + 体验优化。

## 原始 Goal（2026-06-14）

我们需要支持 ssh 设备的文件查看并优化体验：

1. 参考 llm providers 设置，优化文件位置白名单设置的体验，每个文件位置的增加/编辑为单独 modal，
   每个文件位置有单独的启用开关。
2. 文件位置应该绑定到特定设备，本地设备则按照本地文件读取，ssh 设备使用 rsync 处理。
3. 文件 tab 里默认不去监听没有展开的树根；浏览器里需要保存文件树的展开状况，但要处理好权限问题、
   文件不存在问题、设备不存在问题、树根被删除或禁用、连接失败、rsync 未安装等边界情况；允许用
   收起节点再展开节点来做重试，这类操作需要做好防抖。
4. 无论本地还是远程设备，若检测到 rsync 未安装，需给出错误提示。若用户配置过 LLM，则在错误提示中
   加一个「自动安装 rsync」按钮；点击后创建一个窗口并为该窗口跳转到一个新的 agent chat 并预填
   prompt，等待用户手动确认发送。流程需注意时序：保证窗口创建好后再跳转 agent chat，且流程不被
   其他操作打断；手机环境强制打开 sidebar；按钮是一次性行为，触发安装流程后立即清除当前 toast。
5. 无论远程还是本地设备，处理好窗口生命周期和树根生命周期，处理好队列，避免 IO 搞死整个程序。

要求：按项目规范完成 goal，善用 subagent。

## 关键设计判断（待探索后定稿）
- **rsync 作为统一文件传输层**：req4 明确要求「本地与远程」都有 rsync 未安装的检测与自动安装流程，
  这只有在本地文件访问也依赖 rsync 时才成立。故本地设备 = 对本地路径跑 rsync，ssh 设备 = rsync over ssh。
- 每个 file root 绑定 deviceId + enabled 开关。
- 需深挖：SSH 鉴权如何物化为 rsync 的 `-e ssh ...`；gateway 如何跑子进程；tmux 建窗时序；
  agent 会话 draft 预填与跳转时序；并发/队列防 IO 打爆。
