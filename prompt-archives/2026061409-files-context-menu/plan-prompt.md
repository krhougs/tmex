# 文件列表 node 右键/长按菜单 + 拖拽传输（issue #21）— Prompt 存档

## 来源

GitHub issue: https://github.com/krhougs/tmex/issues/21 （标题「文件系统迭代」，已打 `confirmed` 标签）

## Issue 原文

> ### 文件列表重构
> 文件列表的每个node新增右键菜单（手机端为长按菜单）：
> - 对于所有
> 	- 复制绝对位置
> 	- 复制相对位置(相对于树根)
> 	- Send to Agent
> 		- 在对应设备创建一个新的窗口并在agent中预填这个文件/文件夹的绝对位置
> - 对于文件夹
> 	- 展开/收起
> 	- 上传文件到这个文件夹
> 	  - 支持拖拽到文件夹node上
> - 对于文件
> 	- 打开
> 	- 下载（支持把node直接拖拽到系统中）

## 规划阶段交互（关键决策）

规划时以 4 个并行核查 agent 对最不确定的技术点逐项核实（base-ui ContextMenu、浏览器拖放、上传后端反向 rsync、Send-to-Agent 编排复用），结论见 `plan-00.md` 的「关键技术结论」。

用户在规划阶段确认的两个 UI 决策：
1. **下载入口移入右键/长按菜单，移除文件行的内联下载图标按钮**（下载只通过菜单"下载"项 + 拖拽到桌面）。
2. **仅右键(桌面)/长按(移动)触发菜单，不额外加常显的 ⋮ 按钮**（严格按 issue 范围）。
