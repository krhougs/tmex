# Plan 01：修复滚动与恢复设备管理入口

时间：2026-02-11

## 背景

在上一轮重构后，出现两个体验问题：
1. 设置页不可滚动，较长内容无法操作。
2. 侧边栏底部“管理设备”入口被“设置”替换，设备管理首页入口心智被破坏。

## 目标

1. 让设备列表页和设置页都可滚动。
2. 终端页保持当前不滚动行为，避免影响 xterm 交互。
3. 恢复“管理设备”入口，并与“设置”入口并列提供。

## 关键决策

1. 滚动白名单路由：`/`、`/devices`、`/settings`。
2. 终端详情路由（`/devices/:deviceId/windows/:windowId/panes/:paneId`）继续保持 `overflow-hidden`。
3. 侧边栏底部展开态采用双按钮并列；折叠态采用两个图标按钮。
4. 空设备态恢复“添加设备”主入口，并保留“打开设置”辅助入口。

## 实施任务

1. 调整 `RootLayout` 主内容区滚动策略为路由条件控制。
2. 调整 `Sidebar` 底部入口为“管理设备 + 设置”双入口。
3. 调整空设备态入口文案与跳转目标。
4. 运行前端 TypeScript 编译与关键 E2E 进行回归验证。

## 验收标准

1. 访问 `/devices` 与 `/settings` 可正常滚动。
2. 访问终端页时页面不引入额外滚动行为。
3. 侧边栏展开/折叠态均可访问“管理设备”和“设置”。
4. 空设备态可从“添加设备”恢复到设备管理首页流程。
